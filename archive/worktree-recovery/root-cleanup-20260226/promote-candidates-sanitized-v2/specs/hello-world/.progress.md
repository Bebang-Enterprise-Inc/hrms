Analyzed 12 API files in `hrms/api/`. Key findings:

2. **Location**: Create `hrms/api/hello.py`
3. **URL**: `/api/method/hrms.api.hello.hello`
4. **Quality tools**: pre-commit, ruff, bench tests

See `research.md` for full details.

## Requirements Summary

## Design Summary

Single-file implementation: `hrms/api/hello.py`

Two functions:
1. `hello()` - guest-accessible, returns message + timestamp
2. `hello_authenticated()` - requires login, returns personalized message

No database access. No external dependencies. Minimal risk.

See `design.md` for full details.
