# ADR-0005: Ownership Bootstrap - Sam as accountable owner for all architecture domains

**Status:** Accepted  
**Date:** 2026-02-26

## Context

1. SAD audit identified missing ownership matrix as a release blocker.
2. Domain-level owners for architecture, security, reliability, and DR were not explicitly documented.
3. Architecture hardening required immediate single-point accountability.

## Decision

Assign `Sam Karazi` as accountable owner for all architecture domains until delegates are explicitly named in `OWNERSHIP_MATRIX.md`.

## Consequences

1. Architecture decisions can be unblocked without waiting for undefined role assignments.
2. Future delegate assignments must be documented with date, scope, and escalation path.
3. Governance and escalation now have an explicit accountable endpoint.

## Evidence

1. `docs/architecture/OWNERSHIP_MATRIX.md`
2. `docs/architecture/SOLUTION_ARCHITECTURE_DOCUMENT.md`
3. `docs/plans/2026-02-26-sad-v1-execution-plan.md`
